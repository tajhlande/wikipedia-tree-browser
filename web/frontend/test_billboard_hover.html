Billboard Hover Functionality Test

Test Steps:

  1. Open the application in your browser
  2. Navigate to a node view (select a namespace and load some nodes)
  3. Open the browser's developer console (F12 or Ctrl+Shift+I)
  4. Hover your mouse over the billboard labels (text labels next to nodes)
  5. Look for console messages starting with "[INTERACT]"

Expected Behavior:

  • When you hover over a billboard, you should see console messages like:
  • [INTERACT] Pointer move event, picked mesh: billboard_123
  • [INTERACT] Hovering over billboard 123: Node Label
  • A panel should appear in the bottom-left corner showing node information

Troubleshooting:

  • If you don't see any [INTERACT] messages, the pointer events aren't being captured
  • If you see pointer events but no hover messages, the billboard might not be pickable
  • If you see hover messages but no UI panel, check the BillboardInfoOverlay component

Debugging Information:

Check the following in your code:

  1. Billboards should have isPickable = true
  2. Billboards should have proper naming: billboard_${node.id}
  3. Interaction manager should be initialized and receiving pointer events
  4. Nodes should be registered with the interaction manager
